Dutch Bros UX Evaluating the Dutch Bros website

Timeframe
May 2022, 2 months
Team
With Tyler Waite, Frances Weller, & Kimberly Yasutake
What I did
User experience research, UI design
Tools
Figma, Virtual Research Tools (Survey, interview)

Project Goal

Improve users ability to accomplish key tasks on the Dutch Bros website.The focus is on research: What are the key tasks and what prevents successful completion of it?

Dutch Bros. is a drive-through chain with a focus on drinks and especially coffee.

We want to find out the key tasks on the website, both from the users and the company’s perspective. Then, we want to see if users are actually able to perform those key tasks.

Team: (1)Research done in a group, then (2)give design recommendations solo.

Research Methods

After familiarizing ourselves with the website, we set out on 3 main phases of research.

Survey to get a general understanding of users

We targeted coffee drinkers, wanting to discover:

  • Demographics
  • Familiarity with technology and internet usage
  • Familiarity with Dutch bros
  • Key needs for a coffee website

Web Accessibility Analysis to locate potential obstacles

We performed an analysis to discover potential obstacles that may trip users up, in order to guide our next steps.

Standard used: UserFocus Web Accessibility Guidelines. This is a set of 247 guidelines created by UX expert Dr. David Travis.

Usability Testing to verify obstacles discovered above, to potentially uncover more obstacles

Our goal was to see:

  • Can users accomplish key tasks?
  • Do the obstacles that surfaced in the guideline analysis actually trip users up?

The test consisted of 4 scenarios and 17 tasks, and a pre- and post- test questionnaire.

Findings

We learned about our users and also uncovered the two main types of issues that tripped users up.

Personas

We took what we learned about our users from the survey and created personas from it.

Issue #1

Ineffective visual signals hid useful, key features in pain sight

The site has useful, user-centered features that users don’t know are there and don’t use due to errors in visual styling.

  • Non-descriptive (icon-only) quick-access buttons that turned a 10 sec task into one that took an average of 1.5 minutes.
  • No indication for how to do horizontal scrolling on desktop, confusing users and preventing success.
  • Opposite signaling: Some things looked clickable but actually were not.
  • Unobtrusive search bar increased search time and frustrated users, with NONE of the users using the search bar. Instead, they all manually scanned the FAQ- even though a search bar available the whole time!
Issue #2

Unclear and nonintuitive navigation stymied users

The site structure was unclear, making navigation almost impossible.

  • 38% failure rate in navigating between the main site and the separate e-commerce site.
  • Similar information located in completely different places: HQ jobs and barista jobs are listed separately; the ‘careers’ menu tab leads only to barista jobs.

Design Goals

With the problems pinpointed, it was time to formulate goals for the redesign.

Improve users’ ability to (quickly) accomplish key tasks.

The goal was to reduce task time & error rate on the key tasks; however, we weren’t able to actually test and collect data on the new design(s).

Design Recommendations

Here are my recommendations for how the findings could be implemented and resolve the obstacles we discovered. This was completed individually, without team collaboration. The timeframe was 1 week.

Mockups were created in 2 sizes; the desktop size mockups are shown here.

Signal key features, make the function clear with a glance

  • Use descriptive text to label buttons
    (Home: annotation 1, 5, 6)
  • Make affordances bigger and bolder
    (Home: annotation 3; Careers: annotation 4)
  • Remove “clickable” affordance for items that can’t be clicked
    (Snacks: annotation 2)

Add visible controls for novel interactions

As horizontal scrolling is rare for desktop users, add visible button controls to ensure they can view the content (Home: annotation 2)

Clarify site structure and make backtracking easy

  • Show that the main site and the e-commerce site are two different sites by adding a “new window” icon to e-commerce site links and adding a big header to the e-commerce site.
    (Home: annotation 4, 7; Store: annotation 1, 2)
  • Implement a visible backtrack button
    (Store: annotation 2)

Consolidate similar info. Display all job openings on the same page

  • From there, each type of role (barista, internships, or HQ roles) can split into specific details on that type of role. (Careers: annotation 1).
  • Users can also see all job listings at once (Careers: annotation 3)

Reflection

This was my first time working on a UX project with a team, and I really enjoyed the experience.

This project showed difficulties that can arise when integrating 3rd party services as wonky integration were the cause of two major navigation issues (the e-commerce site as well as the careers listings). With some UX and careful thinking about the structure and user flow, the difficulties could be smoothed out with UX.

If I had a chance to work more on this project, I would have wanted to conduct a second round of usability testing—this time on the redesign, to see if the numbers improved.
