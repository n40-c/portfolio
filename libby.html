Team

About the project

This project centers around the app Libby, which is a portal to the digital catalog of partnered libraries, with users able to borrow and puruse library-owned content on the app.

In this project, I'll be looking at one of the main library features: book borrowing. In particular, the experience of searching for a title to borrow.

In a physical library, it's pretty standard to use the library's signage system or the Dewey decimal classification system. But how about in a virtual system? Are users able to navigate through the catalog in Libby and successfully find titles they'd like to borrow?

The short of it: the goal of this project is to evaluate & improve the book discovery process of the library app Libby.

Research + Findings

Who are the users and what do they want? In order to get a picture of users' expectations and desires, I asked 5 participants questions that ranged from general (such as asking about their motivations for borrowing) to specific (such as asking about what types of information they use to decide whether to borrow a title).

After learning expectations and desires, I evaluated the current design for points of confusion. I choose to perform a round of usability testing as it would directly show what users have trouble with. I also did a heuristics analysis using established guidelines for web design (I used the Web Accessibility Guidelines) and science-backed heuristics like Hick’s Law.

This is what I found:

Finding #1

There are 3 main userflows:

  • Picky Reader (Only these participants struggled)
  • Searchers
  • Recommendation Seeker

The usability tests revealed that users tend to look for books using one of the 3 methods below (with only the "Picky Reader" userflow running into frustrations):

  • Picky Reader: these participants know what types of books & combinations of genres they like. However, app does not have robust capabilities for detailed searching and filtering, leading to frustration.
  • Searchers: these participants had a specific book in mind and simply used the search bar.
  • Recommendation Seeker: these participants depended on recommendations such as library-curated guides.
Finding #2

Users slowed by information overload & icon, site structure confusion

  • Users are slowed down by information overload, partially caused by redundant listings.
  • Icon and site structure confusion caused by Libby-unique icons that don’t have a descriptive text label, and site structure that isn’t clear at a glance. Both of these issues surfaced in the heuristics analysis and were validated by the usability tests.
Finding #3

Although respondents varied greatly in genre preferences, desired descriptive info was similar (though lacking from the app).

  • Book preferences varied: Some liked a range of genres, others liked just one. Some knew what they liked and stuck to it, while others were overwhelmed by the choices.
  • Descriptive info lacking: Respondents tended to find similar types of information helpful when deciding whether to borrow a title or not. For instance, all respondents considered genre & user reviews helpful. Critically, these types of info were lacking on the app which had sparse genre-tagging (most titles had only 1-3 tags) and no user review system.

Design Goals

With the problems pinpointed, it was time to formulate goals for the redesign.

How can we help all 3 user flows find titles they'd want to borrow?

In particular, the final design should fulfill the following:

  • implement features targeted towards all 3 user flows
  • address confusion by clarifying site structure, icons, and redundant listings
  • include descriptive info desired by users

Ideation + Design Process

With clear goal in mind, I set off to find possible solutions. I brainstormed features & did research, iterated on the UI design, and made an animated mockup.

Brainstormed Features

I brainstormed potential features, took inspiration from existing design solutions, and listed the strengths and weakness of each potential fix. Finally, I picked the features that would best address the obstacles and opportunities discovered in the research phase.

Iterated UI Design

Next, I set out to implement the features in a UI design. In general, my process was to sketch wireframes by hand to work out the general layout, then create a grayscale mock-up to work out the details, and then to finalize with colors and images. Each stage was an exploration, and sometimes even needed a step back to the previous stage.

Made Motion Study

I also made some motion studies to show the transition animations. I planned out the animation changes on paper, then animated using Adobe After Effects.

Solutions

Here's a walkthrough of the final design and callouts to the key features that address the usability issues discovered previously.

Feature #1

Personalized suggestions

This feature suggests titles based on user history. Users can also fine tune suggestions.

This helps Picky Readers by learning their specific preferences and suggesting titles they might be interested in. These recommendations also address the wants of Recommendation Seekers.

In addition, this feature provides a point of entry and reduces the feeling of overwhelmingness. It should also be comfortable and familiar to users as the feature is common on social media apps.

Feature #2

Public user curated collections

This feature allows users to group titles that fit their specific preferences (for instance, a category more specific than a genre like "Gothic Horror") and share these with other users.

This serves both Recommendation Seekers as well as Picky Readers (as they can share and find titles that fit their taste but not conventional genres).

Feature #3

Fine-grain filters

These filters allows users to apply a detailed combination of filters. A search feature helps users find the filters they want. Perfect for Picky Readers.

Addressing the other userflows with Quick search, Librarian-curated lists

  • For Searchers, the redesign adds quick search to the main page.
  • For Recommendation Seekers, the redesign keeps librarian-curated lists prominent.

Other key changes:

  • Efficient searching: Show key data such as genres in list view. This is key decision-making data.
  • Clarified structure: Updated layout clearly communicates structure, that multiple lists are included.
  • Clarified icons using descriptive text: Replaced platform-specific icons with descriptive text.
  • Choice overload:
    • Collapsed lists into 1 clickable item (instead of listing out list items)
    • Combined format-redundant listings. Added descriptive tags and a format toggle instead.

Reflection

This project highlighted to me the importance of testing with real users. The obstacles and opportunities I discovered during research and analysis were very different from what I had personally expected. Testing ensured that I was addressing real issues.

If I had more time, I would like to create a prototype and conduct another round of usability testing. I also would also have liked to explore more facets of the app such as the dashboard, managing holds/borrows, and so on.
